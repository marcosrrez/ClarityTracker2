# ClarityLog Data Flow Documentation

## Overview
This document outlines the complete data flow connections throughout the ClarityLog application, showing how data moves between components and where integration points exist.

## Core Data Flow Architecture

### 1. User Authentication & Session Management
```
Firebase Auth → useAuth Hook → User Context → All Components
```
- **Entry Point**: Firebase authentication system
- **Integration**: User ID flows to all data queries and storage operations
- **Active Status**: Supervisor interface updates user's active supervisor status in real-time

### 2. Supervisor Management Data Flow

#### Supervisor Creation & Updates
```
SupervisorProfileManager → API Routes (/api/supervisors) → DatabaseStorage → PostgreSQL
```

**Key Integration Points:**
- **Start Date Editing**: Editable date fields update supervisor records via PATCH requests
- **Session Scheduling**: "Schedule Session" button calculates next session dates based on frequency
- **Compliance Status**: "Mark Compliant" updates supervisor compliance in database
- **Dashboard Integration**: Active supervisor status flows to dashboard components

#### Data Flow Chain:
1. **SupervisorProfileManager.tsx** - User interface for supervisor management
2. **server/routes.ts** - API endpoints for supervisor CRUD operations
3. **server/storage.ts** - Database abstraction layer
4. **shared/schema.ts** - Type definitions and validation schemas
5. **PostgreSQL Database** - Persistent data storage

### 3. Dashboard Data Integration

#### AI Integration Tracker
```
AIIntegrationTracker → /api/ai/smart-insights → EnhancedAIService → AI Providers (OpenAI/Google AI)
```
- **Icon**: Now uses Lightbulb instead of Brain for insights
- **Milestone Tracking**: Tracks AI insights, patterns detected, supervision prep generated
- **Progress Calculation**: Calculates percentage based on total insights vs. targets

#### Personalized AI Coaching
```
PersonalizedAICoaching → /api/ai/personalized-insights → AI Services → User Log Entries
```
- **Icon**: Updated to use Lightbulb for therapy profile insights
- **Data Sources**: Combines user logs, therapy profile, and supervisor feedback
- **Contextual Insights**: Generates personalized recommendations based on user progress

### 4. Supervision Session Data Flow

#### Session Scheduling & Tracking
```
Supervisor Interface → Session Creation → Supervision Sessions Table → Dashboard Updates
```

**Integration Chain:**
1. User clicks "Schedule Session" in supervisor interface
2. System calculates next session date based on frequency settings
3. Session record created in supervision_sessions table
4. Dashboard reflects updated session count and compliance status
5. AI systems use session data for supervision prep insights

### 5. Progressive Disclosure System

#### Educational Content Flow
```
Dashboard Components → Progressive Disclosure Service → Educational Content → User Interaction Tracking
```
- **Drill-Down Navigation**: Each dashboard card supports multiple levels of detail
- **Context-Aware Content**: Educational materials adapt based on user progress
- **Interaction Tracking**: System tracks which educational content users access

### 6. AI Intelligence Hub Integration

#### Multi-Provider AI Flow
```
User Actions → Intelligence Hub → [OpenAI, Google AI, Counseling Dataset] → Processed Insights → Dashboard Display
```

**Smart Routing:**
- Primary: OpenAI for complex analysis
- Secondary: Google AI for specific tasks
- Fallback: Counseling Dataset for offline capability

### 7. Data Integration Points

#### Real-Time Updates
1. **Supervisor Status Changes** → Dashboard reflects active supervisor immediately
2. **Session Scheduling** → Updates compliance status across all components
3. **AI Insights Generation** → Triggers milestone celebrations and progress updates
4. **Educational Content Access** → Updates user learning profile and recommendations

#### Cross-Component Data Sharing
- **User Profile**: Shared across all components via context
- **Supervisor Information**: Active supervisor data flows to dashboard and AI systems
- **Log Entries**: Feed into AI analysis, supervision prep, and progress tracking
- **Insights & Patterns**: Generated by AI, displayed in dashboard, used for educational content

## Key Database Tables & Relationships

### Core Tables
1. **users** - User profiles and authentication
2. **supervisors** - Supervisor information and settings
3. **supervision_relationships** - Links users to supervisors with status tracking
4. **supervision_sessions** - Individual session records
5. **log_entries** - User activity and session logs
6. **user_progress_insights** - AI-generated insights and patterns

### Relationship Flow
```
users → supervision_relationships → supervisors
users → log_entries → ai_analysis → user_progress_insights
supervisors → supervision_sessions → compliance_tracking
```

## API Endpoints & Data Flow

### Supervisor Management
- `GET /api/supervisors/:userId` - Fetch user's supervisors
- `POST /api/supervisors` - Create new supervisor relationship
- `PATCH /api/supervisors/:id` - Update supervisor details (including start dates)
- `POST /api/supervisors/:id/schedule-session` - Schedule new supervision session

### AI & Insights
- `GET /api/ai/smart-insights` - Fetch AI-generated insights for dashboard
- `GET /api/ai/personalized-insights` - Get personalized coaching recommendations
- `POST /api/ai/analyze-session` - Analyze individual session data

### Progressive Disclosure
- `GET /api/progress-insights/:userId` - Fetch progress insights for disclosure system
- `POST /api/track-interaction` - Track user educational content interactions

## Real-Time Data Synchronization

### Update Triggers

2. **Session Scheduling** → Updates next session dates and compliance status
3. **Mark Compliant Action** → Updates supervisor compliance across all views
4. **AI Insight Generation** → Triggers milestone checks and celebrations
